secureapp

secureapp is a Linux-focused file encryption tool written in Rust. It provides authenticated encryption using:

  • XChaCha20-Poly1305 (AEAD)
  • Argon2id password-based key derivation
  • HKDF key separation
  • Atomic in-place file replacement

Build

cargo build --release
The compiled binary will be located at: 
target/release/secureapp
You may copy it locally: 
cp target/release/secureapp .

Basic Usage

Encrypt In Place (Recommended)

./secureapp enc --in-place -i file.txt
This will:
  • Prompt for password
  • Create a temporary file
  • Encrypt contents
  • Atomically replace the original file

Decrypt In Place

./secureapp dec --in-place -i file.txt
This restores the original plaintext file using atomic replacement.

Encrypt to Separate Output

./secureapp enc -i file.txt -o file.sec

Delete Original After Encryption

./secureapp enc -i file.txt -o file.sec --delete
This securely overwrites the original file before deleting it.

Decrypt to Separate Output

./secureapp dec -i file.sec -o restored.txt

Adjust Argon2 Security Parameters

Defaults:

  • Memory: 1024 MB
  • Iterations: 3
  • Parallelism: 4
Example with stronger settings: 
./secureapp enc --in-place -i file.txt \
    --memory 2048 \
    --iterations 4 \
    --parallelism 4
Higher memory increases resistance to GPU cracking.

Security Model

  • Authenticated encryption (integrity verified)
  • Per-file random salt
  • Per-file random nonce base
  • Chunked encryption for large files
  • Atomic rename to prevent corruption
  • Directory fsync for crash safety (Linux)

If authentication fails during decryption, the file is not replaced.

Example Workflow

# Encrypt
./secureapp enc --in-place -i secrets.txt

# File is now encrypted

# Decrypt
./secureapp dec --in-place -i secrets.txt
Important:
  • If you forget your password, the data is permanently unrecoverable.
  • Always test decryption before deleting backups.
  • Do not reduce Argon2 memory settings unless necessary.
